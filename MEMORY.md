# AI 资讯自动化流程

## 系统架构

### 核心组件
1. **ai_news_text_only.py** - 文本生成脚本（每日执行）
2. **ai-news-generator.html** - 小红书风格图片生成器（用户手动使用）
3. **feishu-file-sender** - 飞书文件发送技能
4. **定时任务** - 早 8 点、晚 8 点自动执行

### 工作流程
```
定时任务触发 (8:00/20:00)
    ↓
ai_news_text_only.py 执行
    ↓
1. 抓取 10 家媒体最新 AI 新闻
   - 量子位、机器之心、36 氪、智东西、车东西
   - 钛媒体、虎嗅、IT 之家、雷锋网、DeepTech
2. 生成两种格式内容
   - 可粘贴内容（用于 ai-news-generator.html）
   - 小红书文案（标题 + 正文 + 链接 + 话题标签）
3. 保存为 txt 文件到 output/text/
    ↓
feishu-file-sender 发送
    ↓
通过飞书 OpenAPI 发送 txt 文件给用户
```

## 常见问题解决

### 1. 定时任务执行成功但用户没收到
**原因**: session_id 过期
**解决**: 更新定时任务的 session_id 为当前对话的 session

### 2. 发送显示为其他机器人
**原因**: 使用了全局配置而非工作区配置
**解决**: 修改 feishu-file-sender 脚本，优先读取 `agent.json`

### 3. 部分新闻没有链接
**原因**: 备用新闻数据缺少 url 字段
**解决**: 给备用新闻添加默认 URL（媒体首页或分类页）

### 4. 文件链接不可点击
**原因**: txt 文件是纯文本格式
**解决**: 改用飞书文本消息发送（URL 自动识别为超链接）

## 优化方向

1. **链接质量**: 备用新闻使用更具体的链接（分类页/搜索页，而非首页）

### 2026-04-09 下午修复（关键 Bug 修复）

**用户反馈**："我的网络都是好的，为啥你老说有限制呀"

**真正问题**：❌ 不是网络问题，是**代码逻辑问题**！

**Bug 根源**：
```python
# ❌ 旧代码：在当前页导航
page.goto(full_url)  # 进入详情页
page.go_back()       # 返回 → DOM 重新加载，之前获取的链接全部失效！

# ✅ 新代码：用新标签页
detail_page = context.new_page()  # 独立标签页
detail_page.goto(full_url)        # 不影响主页
detail_page.close()               # 关闭即可
```
