## 💡 经验反思

### 问题与解决方案

#### 1. 图片样式不一致问题
**问题**：Python PIL 生成的图片与 HTML 预览效果差异大（字体、圆角、阴影等）

**解决**：
- ✅ 统一使用 HTML + html2canvas 方案
- ✅ 通过 Playwright 自动化操作浏览器生成图片
- ✅ 保证生成图片与用户预览完全一致

**经验**：当需要保证视觉一致性时，优先使用同一渲染引擎（浏览器），而非跨技术方案。

---

#### 2. 用户操作便捷性问题
**问题**：用户反馈上班途中无法用电脑操作 HTML 生成器

**解决**：
- ✅ 实现全自动浏览器自动化流程
- ✅ 图片直接发送到飞书，手机即可发布
- ✅ 用户零操作，只需收到后发布

**经验**：自动化系统应充分考虑用户的实际使用场景，而非假设用户有固定工作环境。

---

#### 3. 浏览器配置文件锁冲突
**问题**：`BrowserType.launch_persistent_context: Failed to create a ProcessSingleton`

**解决**：
```bash
rm -rf /Users/zhouzhenjiang/.copaw/workspaces/5MUwUP/browser/user_data/SingletonLock
pkill -f "Chrome.*user-data-dir.*5MUwUP"
```

**经验**：
- 浏览器自动化前应先清理锁文件
- 无头模式（headless=True）可避免部分冲突
- 添加重试机制处理偶发失败

---

#### 4. 飞书文件发送路径问题
**问题**：Path 对象直接拼接导致 `can't open file` 错误

**解决**：
```python
# 错误
'--file', image_path  # Path 对象

# 正确
'--file', str(image_path)  # 字符串
```

**经验**：调用外部命令时，所有路径参数必须显式转换为字符串。

---

#### 5. 定时任务超时风险
**问题**：默认超时 120 秒，完整流程可能超时

**解决**：
- 优化流程：并行下载图片
- 精简步骤：减少不必要的等待
- 监控执行时间，必要时调整超时设置

**经验**：自动化流程应设置合理的超时时间，并添加进度日志便于排查。

### 可复用的技术框架

#### 浏览器自动化模板
```python
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch(headless=True)
    page = browser.new_page()
    page.goto(f"file://{html_path}")
    
    # 填写表单
    page.fill("selector", content)
    
    # 等待动态内容
    page.wait_for_selector(".success-indicator")
    
    # 下载文件
    with page.expect_download() as download_info:
        page.click("download-button")
    download = download_info.value
    download.save_as(output_path)
    
    browser.close()
```
